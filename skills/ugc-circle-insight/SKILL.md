# UGC 圈层洞察

开展一次基于证据的圈层研究。不只解释概念，还要识别背后的人群、行为、情绪与机会。

## 研究前置

在任何外部搜索或页面访问之前，除非用户已在当前请求中同时确认登录状态和只读授权，否则先询问：

> 开始前，请确认：希望覆盖的 UGC 平台中，哪些已在当前 Chrome/浏览器中登录？是否授权我使用浏览器或 Computer Use 进行只读研究，包括搜索、打开公开内容和查看评论区，但不点赞、评论、关注、收藏、分享或发布？如果尚未登录，请先完成登录，然后回复“已登录并授权”。

等待用户答复。不要复用其他研究任务中的登录或授权信息。如果用户无法或不愿授权原生平台访问，说明缺失的证据，并询问是否接受降级为公开 Web 资料研究；不要静默降级。

## 执行

1. 为当前对象开启独立研究。除非用户明确纳入本次范围，否则不要复用其他对象的来源、结论、关键词或圈层映射。
2. 通过研究前置后，完整读取并遵循 [references/research-method.md](references/research-method.md)。
3. 操作浏览器或应用前，读取并遵循对应的 Browser 或 Computer Use skill；全程保持只读。
4. 并行开展“圈层与情绪”和“活动与玩法”两条搜索线。同步采样有重复证据的圈层黑话、典型句式、梗与评价语，解释实际话外音和使用边界。不得只搜索目标术语；还要从母文化、身份群体和核心实践扩展到圈层原生活动、线上活动、线下聚集场景、平台玩法与品牌案例。玩法建议必须能追溯到活动样本或参与行为证据。
5. 形成交付物前，读取 [references/report-template.md](references/report-template.md)，只使用有证据支持的栏目。
6. 使用用户的语言，先交付速览，再提出 1–2 个由证据触发的策略追问，最后询问：**“你想深挖活动机制、视觉包装，还是品牌沟通？”**
