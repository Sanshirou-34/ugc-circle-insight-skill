## 1. 隔离本次研究

- 在工作笔记中标明当前研究对象和时间范围。
- 来源从 `S01` 开始编号。更换研究对象时新建证据账本，禁止混用。
- 同一请求比较多个对象时，用 `T1`、`T2` 区分，分别管理直接证据。
- 历史案例和模型记忆只能作为假设，不能作为证据。
- 始终区分 `事实观察`、`跨来源模式`、`洞察推断` 和 `策略建议`。
- 协作交接只传递研究对象、范围、访问状态、开放假设和证据账本；任何继承结论都必须能追溯到来源 ID。

## 2. 确定范围与平台

- 选择圈层真实发布和互动的 2–3 个平台。小红书、抖音是常见候选，但不是强制默认。
- 原生平台内容和评论优先于搜索引擎可见度。
- 默认观察近 90 天；更早内容只用于历史脉络，并明确标记。
- 速览阶段，每个可访问的核心平台以 10–20 个有效样本为目标，不为凑数加入无关内容。
- 深挖阶段，核心平台合计以约 30–50 个有效内容或评论样本为目标。连续约 10 个有效样本不再产生新模式时即可停止。
- 活动生态默认观察近 12 个月；更早案例只作为经典母题或演化脉络。速览覆盖 3–6 个有效活动案例，并尽量包含线上与线下、圈层自发与机构组织等不同类型；深挖再扩展到约 8–12 个，优先保证相关性而非数量。
- 通过研究前置后若访问失败，暂停并请用户恢复访问，或明确同意降级来源方案。

## 3. 搜索前建立概念地图

先判断用户输入的是物件、行为、场景、审美、梗、内容形式、身份标签，还是真正的圈层名称。

提出 2–4 个待验证假设：

`表层术语 → 重复实践 → 母文化 → 身份群体 → 情绪/社会功能`

再建立概念星系：同族概念、上下位概念、载体变体、邻接实践和跨圈使用。候选概念应来自平台搜索联想、共现标签、评论和创作者内容，不能只靠记忆列举。

使用用户自称、稳定共现标签、创作者内容生态、共同互动规则、跨平台复现和反例验证关系，并标记为 `核心 / 母文化 / 子圈层 / 同族 / 邻接 / 跨圈 / 待验证`。

比较三类信息：

- **不变量**：在已验证相关概念中持续成立的部分。
- **变量**：随载体、场景、公开程度、移动性、成本或参与门槛变化的部分。
- **不可迁移项**：相关概念的证据无法证明目标对象的部分。

同时建立活动生态地图，不要等到写建议时才临时寻找玩法：

`母文化/身份群体 → 圈层原生活动 → 线下聚集场景 → 线上活动 → 平台原生玩法 → 品牌/机构案例`

活动候选必须从已验证的母文化、身份自称、核心实践、常见聚集地和共现词继续展开。若圈层同时存在在线与线下生活，速览至少各覆盖一个有效案例；如果某一侧没有找到证据，明确写出缺口。

## 4. 双轨搜索，再反向验证

### 圈层与情绪线

1. 搜索原词、别称、错拼和用户实际用语。
2. 向外扩展到行为、物件、角色、人群自称、母文化、同族概念和情绪张力。
3. 组合“什么意思、入门、教程、展示、挑战、推荐、后悔、争议、圈外、为什么”等意图词。
4. 从目标结果追踪共现标签、评论、创作者持续内容和邻接实践。
5. 从每个候选母文化或同族概念反向搜索目标词，只保留经得住双向验证的关系。
6. 主动寻找误用、拒绝归类、跟风、过时说法、争议和圈外采用等反证。
7. 先建立原生平台样本基线，再用公开 Web 搜索发现线索、补充历史或交叉验证。

### 活动与玩法线

不要只把“活动、玩法”接在目标术语后搜索。先用概念地图找到母文化、身份群体和核心实践，再组合搜索：

`[母文化/身份自称/核心实践] × [活动场景] × [参与动作] × [规则/案例/复盘/吐槽]`

活动场景按圈层实际语言扩展，至少检查以下四类：

- **圈层原生活动**：成员自发形成的仪式、聚会、展示、交换、应援、创作或纪念行为。
- **线下聚集场景**：展、节、赛、会、市集、快闪、主题店、社团、场馆或其他真实聚集地。
- **线上与平台玩法**：征集、挑战、共创、接力、打卡、投票、榜单、合拍、模板、评论区互动或虚拟活动。
- **品牌与机构案例**：平台、IP、展会、商家或品牌已举办的活动，以及参与者对规则和体验的反馈。

例如研究“痛屋”时，活动搜索不应停留在“痛屋活动”，还要沿“痛文化/二次元/ACG”继续搜索漫展、同人展、谷子市集、Cos 集会、主题快闪、应援、生贺、线上征集和二创挑战等候选场景，再用参与者内容验证哪些与目标对象真正相关。

每个活动案例提取：组织者、目标人群、线上/线下场景、核心参与动作、产出内容、规则、激励或认可、社交关系、传播回路、可见参与反馈、摩擦点与复用条件。官方页面可以证明规则，不能单独证明参与者喜欢；对情绪与效果的判断必须补充参与者 UGC、评论或复盘。

相邻圈层的活动只能标记为 `跨圈参考`，用于启发机制，不能伪装成目标圈层已经验证的偏好。优先寻找目标圈层反复出现的参与动作，而不是只收集看起来热闹的案例。

登录页面、评论区、视觉模式和搜索联想优先使用专用 Browser 能力；只有更具体的浏览器能力无法读取界面时才使用 Computer Use。不得绕过登录、验证码、反爬、付费墙或安全警告。

## 5. 建立证据账本

每个来源使用一行：

`S01 [平台] [原帖|评论|活动页|复盘|摘要|转载|媒体] [洞察|玩法] [直证|同族|母文化|邻接|反证] [活动:不适用|圈层原生|线下场景|平台玩法|品牌案例|跨圈参考] [日期] [强|中|弱]`

记录链接、可见日期、内容形式、关键语言或视觉、互动行为、可见数据和一句简短观察。不保留用户名、个人信息或大段原文。

有效样本至少揭示一种信息：典型表达、有意义互动、日常用法、传播路径、参与动作、活动规则、参与反馈、摩擦点、争议或反例。在条件允许时覆盖头部、近期增长、日常、争议和反例样本。

信号规则：

- **强**：至少两个相互独立的原生来源重复出现，并有评论或行为证据支持。
- **中**：多个线索方向一致，但样本有限、来源较弱或只在单一平台成立。
- **弱**：单条爆款、单个摘要、单篇解释文章或尚未复现的新说法。

同族和母文化证据可以解释目标对象，但不能单独把目标结论升级为强信号。分别报告样本数，不要把不同平台的绝对互动量当成同一指标比较。

活动存在不等于活动有效。规则是否成立、参与是否发生、情绪是否被满足是三种不同证据；分别标记，不用主办方宣传替代参与者反应。

## 6. 形成洞察与玩法

每个重要模式都写出以下证据链：

`内容信号 → 可见互动 → 可能情绪 → 替代解释 → 设计启示 → 参与风险`

覆盖概念边界、当前脉搏、语言符号、互动习惯、内部张力以及过时或有争议的用法。情绪只能写成有依据的假设，不能写成心理诊断。

从品牌能够可信服务的层级寻找机会，如容器、工具、舞台、仪式、认可或连接。不要只复制表层符号而忽略底层需求。

先从活动案例抽取可复用机制：

`活动案例 → 已验证参与动作 → 情绪/社会回报 → 互动关系 → 传播回路 → 摩擦点 → 可复用条件`

再把机制拆成玩法原子：

`场景 + 触发方式 + 参与动作 + 内容产出 + 他人互动 + 激励/认可 + 扩散回路 + 参与门槛`

用圈层已经表现出的行为证据重组玩法，不从通用营销清单随机抽取。方向之间必须在参与动作或关系结构上真正不同，而不只是更换主题和视觉。每个建议明确：继承了哪些已有机制、做了什么改造、为什么适合当前圈层、哪些部分尚待验证。

## 7. 通过追问深挖

速览后提出能够暴露真实策略分叉的问题：要服务哪一层人群、回应哪种底层需求、品牌承担什么角色、活动要激发什么行为。每次询问 1–3 个，不重复已有答案。

深挖同一对象时沿用证据账本并追加来源；研究对象发生实质变化时，新开一次独立研究。
