# 研究方法

## 1. 隔离本次研究

- 在工作笔记中标明当前研究对象和时间范围。
- 来源从 `S01` 开始编号。更换研究对象时新建证据账本，禁止混用。
- 同一请求比较多个对象时，用 `T1`、`T2` 区分，分别管理直接证据。
- 历史案例和模型记忆只能作为假设，不能作为证据。
- 始终区分 `事实观察`、`跨来源模式`、`洞察推断` 和 `策略建议`。
- 协作交接只传递研究对象、范围、访问状态、开放假设和证据账本；任何继承结论都必须能追溯到来源 ID。

## 2. 确定范围与平台

- 选择圈层真实发布和互动的 2–3 个平台。小红书、抖音是常见候选，但不是强制默认。
- 原生平台内容和评论优先于搜索引擎可见度。
- 默认观察近 90 天；更早内容只用于历史脉络，并明确标记。
- 速览阶段，每个可访问的核心平台以 10–20 个有效样本为目标，不为凑数加入无关内容。
- 深挖阶段，核心平台合计以约 30–50 个有效内容或评论样本为目标。连续约 10 个有效样本不再产生新模式时即可停止。
- 通过研究前置后若访问失败，暂停并请用户恢复访问，或明确同意降级来源方案。

## 3. 搜索前建立概念地图

先判断用户输入的是物件、行为、场景、审美、梗、内容形式、身份标签，还是真正的圈层名称。

提出 2–4 个待验证假设：

`表层术语 → 重复实践 → 母文化 → 身份群体 → 情绪/社会功能`

再建立概念星系：同族概念、上下位概念、载体变体、邻接实践和跨圈使用。候选概念应来自平台搜索联想、共现标签、评论和创作者内容，不能只靠记忆列举。

使用用户自称、稳定共现标签、创作者内容生态、共同互动规则、跨平台复现和反例验证关系，并标记为 `核心 / 母文化 / 子圈层 / 同族 / 邻接 / 跨圈 / 待验证`。

比较三类信息：

- **不变量**：在已验证相关概念中持续成立的部分。
- **变量**：随载体、场景、公开程度、移动性、成本或参与门槛变化的部分。
- **不可迁移项**：相关概念的证据无法证明目标对象的部分。

## 4. 向外搜索，再反向验证

1. 搜索原词、别称、错拼和用户实际用语。
2. 向外扩展到行为、物件、角色、人群自称、母文化、同族概念和情绪张力。
3. 组合“什么意思、入门、教程、展示、挑战、推荐、后悔、争议、圈外、为什么”等意图词。
4. 从目标结果追踪共现标签、评论、创作者持续内容和邻接实践。
5. 从每个候选母文化或同族概念反向搜索目标词，只保留经得住双向验证的关系。
6. 主动寻找误用、拒绝归类、跟风、过时说法、争议和圈外采用等反证。
7. 先建立原生平台样本基线，再用公开 Web 搜索发现线索、补充历史或交叉验证。

登录页面、评论区、视觉模式和搜索联想优先使用专用 Browser 能力；只有更具体的浏览器能力无法读取界面时才使用 Computer Use。不得绕过登录、验证码、反爬、付费墙或安全警告。

## 5. 建立证据账本

每个来源使用一行：

`S01 [平台] [原帖|评论|摘要|转载|媒体] [直证|同族|母文化|邻接|反证] [概念层级] [样本角色] [日期] [强|中|弱]`

记录链接、可见日期、内容形式、关键语言或视觉、互动行为、可见数据和一句简短观察。不保留用户名、个人信息或大段原文。

有效样本至少揭示一种信息：典型表达、有意义互动、日常用法、传播路径、争议或反例。在条件允许时覆盖头部、近期增长、日常、争议和反例样本。

信号规则：

- **强**：至少两个相互独立的原生来源重复出现，并有评论或行为证据支持。
- **中**：多个线索方向一致，但样本有限、来源较弱或只在单一平台成立。
- **弱**：单条爆款、单个摘要、单篇解释文章或尚未复现的新说法。

同族和母文化证据可以解释目标对象，但不能单独把目标结论升级为强信号。分别报告样本数，不要把不同平台的绝对互动量当成同一指标比较。

## 6. 形成洞察

每个重要模式都写出以下证据链：

`内容信号 → 可见互动 → 可能情绪 → 替代解释 → 设计启示 → 参与风险`

覆盖概念边界、当前脉搏、语言符号、互动习惯、内部张力以及过时或有争议的用法。情绪只能写成有依据的假设，不能写成心理诊断。

从品牌能够可信服务的层级寻找机会，如容器、工具、舞台、仪式、认可或连接。不要只复制表层符号而忽略底层需求。

## 7. 通过追问深挖

速览后提出能够暴露真实策略分叉的问题：要服务哪一层人群、回应哪种底层需求、品牌承担什么角色、活动要激发什么行为。每次询问 1–3 个，不重复已有答案。

深挖同一对象时沿用证据账本并追加来源；研究对象发生实质变化时，新开一次独立研究。
