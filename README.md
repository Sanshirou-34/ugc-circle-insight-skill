# UGC 圈层洞察 Skill

帮助品牌与设计团队从当前 UGC 内容、评论、互动行为和既有活动案例中，快速理解陌生圈层的概念、情绪、黑话、互动习惯与活动机会。

它不会停留在术语释义，而会从表层词汇继续下钻到核心实践、母文化、身份群体和关联概念。例如研究「痛屋」时，会同时追溯痛文化、二次元群体，以及痛包、痛车、痛娃等概念家族。

## 适用场景

- 快速进入陌生圈层或亚文化
- 为 UGC 活动寻找机制与参与动机
- 提炼圈内黑话、情绪张力和互动习惯
- 为视觉包装与品牌沟通提供依据

## 使用方式

在 Codex 中发送：

```text
请从 https://github.com/Sanshirou-34/ugc-circle-insight-skill/tree/main/skills/ugc-circle-insight 安装 ugc-circle-insight skill
```

安装后可以这样提问：

```text
使用 ugc-circle-insight 调研「痛屋」。
```

Skill 会先确认小红书、抖音等 UGC 平台的登录与授权状态，再开始研究；若平台无法访问，会明确标记证据缺口并降级使用公开网页信息。

## 研究节奏

默认先给出一版可快速阅读的圈层速览，再询问：

> 你是想做活动机制、视觉包装，还是品牌沟通？

用户选择方向后，再继续深挖。输出同时覆盖洞察和建议，但以活动玩法为主。

## 研究方法预览

完整版本见 [`research-method.md`](skills/ugc-circle-insight/references/research-method.md)。核心研究流程如下：

1. **隔离研究对象**：为每次研究单独建立证据账本，以 `S01`、`S02` 编号，避免不同圈层或历史记忆相互污染。
2. **确认范围与平台**：选择圈层真实发布和互动的 2–3 个平台；默认观察近 90 天，原生 UGC 内容与评论优先。
3. **建立概念地图**：先判断用户输入的是物件、行为、场景、审美、梗，还是真正的圈层名称，再向母文化和身份群体下钻。
4. **开展双轨搜索**：一条研究圈层与情绪，一条扫描圈层原生活动、线下场景、线上/平台玩法和品牌案例；同时寻找误用、争议与失败反馈等反证。
5. **记录证据强度**：区分直证、同族、母文化、邻接与反证，并将信号标记为强、中、弱。
6. **形成证据链**：从可见内容与互动出发推断情绪，从已有活动中提炼参与动作和传播机制，再转化为设计启示与玩法方向。
7. **通过追问深挖**：速览后围绕目标人群、底层需求、参与行为和品牌角色继续研究。

研究不会只匹配用户给出的关键词，而是使用下面的结构寻找更底层的圈层关系：

```text
表层术语 → 重复实践 → 母文化 → 身份群体 → 情绪 / 社会功能
```

例如研究「痛屋」时，不只搜索痛屋，还会验证它与痛文化、二次元群体、痛包、痛车、痛娃等概念之间的关系，并区分核心、同族、邻接和跨圈使用。

每条重要洞察都需要形成证据链：

```text
内容信号 → 可见互动 → 可能情绪 → 替代解释 → 设计启示 → 参与风险
```

证据账本中的每个来源都会带有标签：

```text
S01 [平台] [原帖|评论|活动页|复盘|摘要|转载|媒体]
    [洞察|玩法] [直证|同族|母文化|邻接|反证]
    [语言:不适用|黑话|缩写|梗|句式|评价|自嘲|争议]
    [活动:不适用|圈层原生|线下场景|平台玩法|品牌案例|跨圈参考]
    [日期] [强|中|弱]
```

### 黑话与典型语录采样

Skill 会从原生帖子、视频文案和评论对话中寻找反复出现的黑话、缩写、梗、评价语、自嘲和句式模板，并观察圈内人如何接话、解释、纠正或反对某种用法。

每个候选表达都会经过下面的翻译链路：

```text
短语或短句 → 类型 → 字面意思 → 通俗解释 / 实际话外音
→ 使用场景与关系 → 语气 / 情绪 → 流行状态 → 圈外误用风险
```

单条爆款文案、个人口头禅或一次性情绪不会被包装成圈层共识。语录只保留解释所需的短句，长表达使用转述，并给出品牌使用边界。

### 活动生态扫描

玩法研究不会只把“活动”接在目标术语后面搜索，而会先找到母文化、身份群体和核心实践，再组合：

```text
[母文化 / 身份自称 / 核心实践]
× [活动场景]
× [参与动作]
× [规则 / 案例 / 复盘 / 吐槽]
```

默认检查四类活动证据：

- 圈层成员自发形成的仪式、聚会、展示、交换、应援或创作；
- 展、节、赛、会、市集、快闪、主题店等线下聚集场景；
- 征集、挑战、共创、接力、打卡、投票、合拍、模板等线上或平台玩法；
- 平台、IP、展会、商家或品牌案例，以及参与者对规则和体验的反馈。

例如研究「痛屋」时，会沿痛文化、二次元和 ACG 继续搜索漫展、同人展、谷子市集、Cos 集会、主题快闪、应援、生贺、线上征集与二创挑战，再反向验证哪些机制与痛屋真正相关。

活动案例会被拆成可重组的玩法原子：

```text
场景 + 触发方式 + 参与动作 + 内容产出
+ 他人互动 + 激励 / 认可 + 扩散回路 + 参与门槛
```

## 报告模板预览

完整版本见 [`report-template.md`](skills/ugc-circle-insight/references/report-template.md)。速览报告默认包含：

### 1. 一句话圈内翻译

用圈内人能够认可、圈外人能够理解的方式解释术语，同时说明它不是什么，以及它属于主流、新兴、小众、争议、过时还是待验证用法。

### 2. 圈层地图与概念星系

```text
表层术语 → 核心实践 → 母文化 → 身份群体 → 情绪 / 社会功能
```

| 概念或层级 | 关系 | 共享机制 | 特有变量 | 信号 | 证据 |
|---|---|---|---|---|---|
| 目标术语 | 核心 / 母文化 / 同族等 | 跨概念不变量 | 目标特有差异 | 强 / 中 / 弱 | S01、S02… |

### 3. 当前脉搏

列出 3–5 个当前模式，分别说明信号强度、来源、可见事实和当下意义，并区分长期文化与短期热点。

### 4. 圈层黑话与典型语录

| 黑话或短句 | 类型 | 字面意思 | 通俗解释：实际在表达什么 | 使用场景与关系 | 语气/情绪 | 状态 | 品牌使用边界 | 证据 |
|---|---|---|---|---|---|---|---|---|

输出优先收录 5–10 条有重复证据、最有助于理解圈层的表达，并区分稳定、流行、新兴、争议、过时和待验证用法。

### 5. 视觉符号与互动习惯

| 信号 | 圈内用法或行为 | 可能情绪 | 替代解释 | 误用风险 | 证据 |
|---|---|---|---|---|---|

### 6. 圈层张力

指出最重要的 1–3 个圈内边界、矛盾或争议，以及它们如何影响参与意愿。

### 7. 已有活动生态与玩法母题

先展示 3–6 个最相关的活动案例，区分规则、实际参与和参与者反应：

| 活动或场景 | 类型 | 线上/线下 | 核心参与动作 | 情绪回报 | 传播回路 | 参与反馈与摩擦 | 可借 / 不可照搬 | 证据 |
|---|---|---|---|---|---|---|---|---|

### 8. UGC 活动机会

每个活动方向都会说明参考了哪些活动母题、继承与改造了什么，并包含参与路径、情绪杠杆、传播或共创钩子、品牌角色、参与门槛、翻车风险和小规模验证方式。

```text
已有活动证据 → 已验证参与动作 → 情绪回报 → 机制改造 → UGC 扩散
```

### 9. 证据、局限与策略追问

报告会公开研究日期、平台访问状态、样本数量、来源链接与降级方案，并根据证据提出 1–2 个真正影响策略选择的问题，最后询问：

> 你想深挖活动机制、视觉包装，还是品牌沟通？

## 数据边界

只使用当前可访问的公开内容或用户已授权访问的页面，不绕过登录、验证码、访问限制或平台安全机制。研究结论会按来源类型标记，并区分事实、观察、推断和建议，避免把单条内容误判为圈层共识。
